QR USAGE AT PHU
① DISPATCH STAFF
Generate QR at DMS warehouse. Download PNG and/or print physical receipt. Attach to shipment or share with driver.
② DRIVER CARRIES
Driver transports ITNs and physical/digital QR receipt to the target PHU facility.
③ PHU STAFF SCANS
PHU in-charge uses phone camera to scan QR. Reads district, chiefdom, PHU, ITN quantities, driver name and verification code to confirm delivery.
Verification Code: Last 6 characters of the Dispatch ID (e.g. 4312). PHU can verify verbally even without a scanner.

GOOGLE SHEETS SYNC ARCHITECTURE
DMS WEB APP
Record saved locally in browser localStorage. Dispatch data ready to push.
→
PENDING QUEUE
If sync fails (offline / timeout), record added to local pending queue. Retried on next save or manual retry.
→
GOOGLE SHEETS
Apps Script /exec webhook receives POST request, appends row to master sheet with all 19 dispatch fields.
Setup: In the HTML source, replace YOUR_SCRIPT_ID_HERE in the GOOGLE_SHEETS_WEBHOOK constant with your deployed Google Apps Script URL (ends in /exec).
Offline Mode: The app works fully offline. Records are stored in localStorage. When connectivity returns, use "↺ RETRY PENDING SYNC" to push queued dispatches to Google Sheets.

KEY FEATURES & TECHNICAL NOTES

Security & Data Integrity

Masked username fields — both staff login and driver verification use password-type inputs with show/hide toggle
Duplicate PHU protection — warning shown and Save button disabled if PHU was already dispatched
Vehicle plate validation — exactly 3 uppercase letters + 3 digits required (ABC-123 format)
Role-based access — DMS Staff see only their district's data; only "All Districts" users see nationally
localStorage safety wrapper — falls back to in-memory store if browser blocks localStorage (e.g. Safari ITP)
Note: Driver usernames are never displayed in dispatch records — only the driver's full name and phone number appear, protecting credential confidentiality.

Offline & Performance

Fully offline-capable — all records saved to localStorage/memory immediately on save, before sync attempt
10-second sync timeout — AbortController cancels hanging requests; fallback to no-cors mode
QR generation is async — Promise-based flow ensures QR is fully rendered before form reset and record is stored
recordsMap lookup — dispatch records indexed by ID for O(1) QR modal retrieval; avoids JSON.stringify in onclick attributes
Sync log capped at 200 entries — prevents unbounded localStorage growth over long campaigns
File Dependencies: Place users.csv, phu_hierarchy.csv, and optionally icf-sl.jpg (logo) in the same folder as the HTML file. The app loads them automatically on startup.
